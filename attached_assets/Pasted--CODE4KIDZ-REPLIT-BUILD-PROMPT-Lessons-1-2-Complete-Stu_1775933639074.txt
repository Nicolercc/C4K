## WHAT YOU ARE BUILDING

A working React + TypeScript web app called Code4Kidz.
This is the Duolingo for coding — a gamified, mascot-guided browser app that teaches
kids ages 7-10 to write real HTML and CSS through short, animated lessons.

You are building the COMPLETE student experience from the moment they land on the
onboarding screen through finishing Lesson 1 and completing Lesson 2.

Every piece of copy, every Byte quote, every validator, every animation state,
and every transition is specified in this document. Do not improvise.
If something is not specified, ask before building.

- React 18 + TypeScript (strict mode)
- Vite (not CRA)

## GLOBAL STATE — gameStore.ts

Build this FIRST. Every other file depends on it.

```typescript
// store/gameStore.ts
import { create } from 'zustand'
import { persist } from 'zustand/middleware'

export type MascotMood = 'idle' | 'cheer' | 'think' | 'sad' | 'story'

export interface MistakeEntry {
  stepId: string
  failCount: number
  instruction: string
  startingCode: string
}

interface GameState {
  // Identity
  topicName: string          // Set during onboarding. Used in ALL Byte lines.
  classroomCode: string      // Optional — set if facilitator gave a code

  // Progress
  currentLessonId: string    // 'lesson-01' | 'lesson-02' etc
  completedLessons: string[] // Array of completed lesson IDs
  currentStepIndex: number   // 0 = warm-up, 1-5 = challenge steps

  // Gamification
  xp: number
  hearts: number             // 0-3, refills between lessons
  streak: number
  lastPlayedDate: string     // ISO date string

  // Lesson state
  code: string               // Current editor value
  failCount: number          // Resets on each new step
  hintsUsed: number          // Resets on each new step
  mascotMood: MascotMood
  byteMessage: string        // Current speech bubble text

  // Mistake tracking (session only — NOT persisted)
  mistakeLog: Record<string, MistakeEntry[]>

  // Actions
  setTopic: (name: string) => void
  setClassroomCode: (code: string) => void
  gainXP: (amount: number) => void
  loseHeart: () => void
  refillHearts: () => void
  advanceStep: () => void
  resetToStep: (index: number) => void
  updateCode: (code: string) => void
  incrementFail: () => void
  resetFail: () => void
  setMascotMood: (mood: MascotMood, message: string) => void
  recordMistake: (lessonId: string, entry: MistakeEntry) => void
  clearMistakeLog: (lessonId: string) => void
  markLessonComplete: (lessonId: string) => void
  checkAndUpdateStreak: () => void
}

// Persist everything EXCEPT mistakeLog (session-only, privacy by design)
export const useGameStore = create<GameState>()(
  persist(
    (set, get) => ({
      topicName: '',
      classroomCode: '',
      currentLessonId: 'lesson-01',
      completedLessons: [],
      currentStepIndex: 0,
      xp: 0,
      hearts: 3,
      streak: 0,
      lastPlayedDate: '',
      code: '',
      failCount: 0,
      hintsUsed: 0,
      mascotMood: 'idle',
      byteMessage: '',
      mistakeLog: {},

      setTopic: (name) => set({ topicName: name }),
      setClassroomCode: (code) => set({ classroomCode: code }),

      gainXP: (amount) => set((s) => ({ xp: s.xp + amount })),

      loseHeart: () => set((s) => ({
        hearts: Math.max(0, s.hearts - 1),
        mascotMood: 'sad',
        byteMessage: s.hearts === 1
          ? "That was your last heart. Take a breath — you can try again tomorrow with full hearts!"
          : "Oops! That is okay. Every coder gets this wrong sometimes."
      })),

      refillHearts: () => set({ hearts: 3 }),

      advanceStep: () => set((s) => ({
        currentStepIndex: s.currentStepIndex + 1,
        failCount: 0,
        hintsUsed: 0,
        code: '',
      })),

      resetToStep: (index) => set({ currentStepIndex: index, failCount: 0, hintsUsed: 0 }),

      updateCode: (code) => set({ code }),

      incrementFail: () => set((s) => ({ failCount: s.failCount + 1 })),

      resetFail: () => set({ failCount: 0, hintsUsed: 0 }),

      setMascotMood: (mood, message) => set({ mascotMood: mood, byteMessage: message }),

      recordMistake: (lessonId, entry) => set((s) => ({
        mistakeLog: {
          ...s.mistakeLog,
          [lessonId]: [...(s.mistakeLog[lessonId] ?? []), entry]
        }
      })),

      clearMistakeLog: (lessonId) => set((s) => {
        const log = { ...s.mistakeLog }
        delete log[lessonId]
        return { mistakeLog: log }
      }),

      markLessonComplete: (lessonId) => set((s) => ({
        completedLessons: [...s.completedLessons, lessonId],
        currentStepIndex: 0,
        hearts: 3,         // refill hearts between lessons
        failCount: 0,
        hintsUsed: 0,
        code: '',
      })),

      checkAndUpdateStreak: () => {
        const today = new Date().toDateString()
        const { lastPlayedDate, streak } = get()
        if (lastPlayedDate === today) return
        const yesterday = new Date(Date.now() - 86400000).toDateString()
        const maintained = lastPlayedDate === yesterday
        set({
          streak: maintained ? streak + 1 : 1,
          lastPlayedDate: today,
        })
      },
    }),
    {
      name: 'code4kidz-store',
      partialize: (s) => {
        const { mistakeLog, ...rest } = s  // NEVER persist mistakeLog
        return rest
      }
    }
  )
)
```

### Validator Logic — validator.ts

```typescript
// utils/validator.ts
import type { LessonStep } from '../data/lessons/lesson-01'

export type ValidationResult = 'pass' | 'fail' | 'error'

export function validate(
  iframeRef: React.RefObject<HTMLIFrameElement>,
  step: LessonStep
): ValidationResult {
  try {
    const doc = iframeRef.current?.contentDocument
    if (!doc) return 'error'
    if (step.type === 'warmup') return 'pass'  // warm-up always passes after 1 attempt
    return step.validate(doc) ? 'pass' : 'fail'
  } catch {
    return 'fail'   // NEVER crash on bad kid code
  }
}
```

### Validation Event Flow — Wire This Exactly

```
ON EVERY KEYSTROKE (debounced 150ms):
  1. Update gameStore.code
  2. Update iframe.srcDoc (the preview re-renders)
  3. After 200ms (give iframe time to parse): run validate()

  IF result === 'pass':
    → Play correct-ping.mp3 (Howler, volume 0.6)
    → Set editor validation bar to green "✓ Perfect!"
    → Byte mood: 'cheer'
    → Byte message: step-specific cheer line (see below)
    → gainXP(failCount === 0 ? step.xp : Math.floor(step.xp / 2))
    → recordMistake if failCount > 0
    → setTimeout(advanceStep, 1200)   ← 1.2 second pause before next step
    → If this was the last step (step-5): navigate to /review/[lessonId]

  IF result === 'fail':
    → NO immediate feedback (avoid rapid red flashing while kid is typing)
    → Only show fail state if code has been unchanged for 2 seconds
    → incrementFail()
    → If failCount >= 2: reveal hint button (Framer Motion slide-up)
    → If failCount >= 1: Byte mood → 'think', byteMessage → think line
    → If hearts > 0 AND failCount is a multiple of 3: loseHeart()
    → Play wrong-buzz.mp3 (Howler, volume 0.4) — only on heart loss, not every fail
```

## CRITICAL RULES — READ THESE BEFORE BUILDING

1. THE EDITOR LOOP IS EVERYTHING.
   CodeMirror → onChange → gameStore.updateCode → iframe.srcDoc update → validate().
   Get this working on Day 1 before anything else. Nothing else matters until
   a kid can type <h1>hello</h1> and see it appear in the preview.

2. NEVER BLANK THE PREVIEW.
   If the HTML is invalid, show whatever parsed. Partial > blank.
   The visual cause-and-effect IS the learning. Do not break it.

3. BYTE'S MESSAGES ARE NOT OPTIONAL.
   Every step has exact bytePrompt copy. Use it verbatim.
   Do not improvise Byte's dialogue. These words were written for a reason.

4. DEBOUNCE IS 150ms, NOT MORE, NOT LESS.
   Too fast = CPU thrash on old hardware.
   Too slow = preview feels laggy and disconnected.

5. HEART LOSS IS THROTTLED.
   A kid can type wrong code for 30 seconds straight — they should not lose
   3 hearts in 30 seconds. Hearts only decrease when the code has been
   invalid and UNCHANGED for 3+ seconds. Not on every keystroke.

6. WARM-UP STEPS NEVER LOSE HEARTS.
   type === 'warmup' → loseHeart() is never called. Period.

7. REVIEW STEPS NEVER LOSE HEARTS.
   ReviewPage → loseHeart() is never called. Period.

8. topicName IS INJECTED INTO BYTE'S MESSAGES.
   Anywhere Byte says "your topic" in general — replace with the actual topicName.
   Example: "Let us build your [topicName] page!" — pull from gameStore.

9. FONTS ARE LARGE. ALWAYS.
   Editor: 18px minimum. Instructions: 16px minimum.
   These are non-negotiable for ages 7-10.

10. ALL ANIMATIONS USE FRAMER MOTION.
    No raw CSS animations except the idle breathing loop on Byte.
    Framer Motion handles: page transitions, Byte mood swaps,
    XP pops, heart loss shakes, speech bubble entrances.

✓ Kid opens /onboarding, enters a topic name, reaches /map
✓ Kid clicks Lesson 1 node, sees all 3 panels
✓ Kid types code in the editor, preview updates in real time

✓ All 5 Lesson 2 steps validate correctly

✓ XP persists across both lessons (localStorage via Zustand persist)
✓ Streak increments if this is the kid's second day
