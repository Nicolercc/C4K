FIX 3 — STEP 5 AUTO-PASS BUG (CRITICAL)
═══════════════════════════════════════════

Current behavior:
The 'combine' step (step 5 of every lesson) pre-fills code that 
already satisfies the validator. The step can pass before the kid 
edits anything, because the startingCode renders in the iframe and 
the validator fires on the initial render.

This is the single most important fix in this prompt.
The kid must earn step 5. It cannot auto-complete.

Fix:

In the validation logic, add a new concept: editRequired.

Add a boolean field to LessonStep:
  editRequired?: boolean

Set editRequired: true on every step with type === 'combine'.

In the validate() function, before checking the step's validator:

  if (step.editRequired && !state.hasEditedCurrentStep) {
    return 'fail'  // Block the pass until the kid types something
  }

Add to gameStore:
  hasEditedCurrentStep: boolean  // resets to false on each step load
  setHasEdited: () => void       // called on first onChange in editor

In Editor.tsx onChange handler:
  On the FIRST keystroke of any step, call setHasEdited().
  This unlocks the validator for editRequired steps.

Result: 
  - The combine step loads with correct-looking code in the editor
  - The preview shows the page with their topic (looks great)
  - But the step does NOT pass yet
  - Byte says: "Look — your whole page is here. NOW make it yours."
  - The moment the kid types a single character: hasEdited becomes true
  - On the next validation cycle: the full validator runs
  - If the h1 still has text: it passes
  - The kid feels like THEY did something
