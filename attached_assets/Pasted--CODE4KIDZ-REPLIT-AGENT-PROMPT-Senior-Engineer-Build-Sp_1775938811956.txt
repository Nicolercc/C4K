```typescript
// src/components/Byte/ByteAnimator.tsx

interface ByteAnimatorProps {
  mood: 'idle' | 'cheer' | 'think' | 'sad' | 'story'
  isTyping: boolean      // true when editor has changed in last 2s
  isStuck: boolean       // true when kid has been on same step 20+ sec without typing

  justFailed: boolean    // true for 1s after validator returns fail
```

### (The most expressive part of Byte)

Byte's SVG has two circular eyes. Each eye has:
- An outer circle (the "white" of the eye)
- An inner circle (the "pupil")
- An eyelid (a rectangle that drops from above to cover the eye)

```

  Eyelid height: 0% → 100% → 0%
  Duration: 0.15s close, 0.08s hold, 0.15s open

ADD: DOUBLE BLINK (surprised reaction):
  Two blinks in rapid succession: close-open-close-open
  Total duration: 0.7s
  Trigger: when justFailed becomes true

ADD: SLOW BLINK (thinking):
  Eyelid height: 0% → 70% → 70% → 0%
  Duration: 0.3s close, 1.2s hold half-closed, 0.3s open
  Frequency: every 6-8s during think mood
  → Byte looks like they are really concentrating

ADD: HAPPY SQUINT (joy):
```

## REACTIVE MOMENTS
### (One-shot animations triggered by events — not loops)

```
ON CORRECT ANSWER (justPassed = true):
  Sequence:
  1. JUMP (y: 0 → -20 → 0, 0.5s, spring)
  2. STAR BURST: 6-8 small yellow/gold circles emit from Byte's center
     Each circle: scale 0 → 1, then moves outward 40-80px in a random direction
     then opacity 1 → 0, total duration 0.8s
     Use Framer Motion with custom variants per particle
  3. HAPPY SQUINT eyes during jump peak
  4. ARMS UP
  5. After landing: 2 more smaller bounces (y: 0 → -8 → 0, 0.3s each)

ON WRONG ANSWER (justFailed = true):
  Sequence:
  1. SHAKE: x: [0, -8, 8, -6, 6, -4, 4, 0], duration 0.5s
  2. DOUBLE BLINK
  3. Small "puff" effect: a cloud shape appears briefly (opacity 0 → 0.6 → 0)
     positioned above Byte's head, scale 0.5 → 1.2, duration 0.6s
  4. Settle into think pose

ON HINT REVEALED:
  1. Byte tilts toward the kid (rotate: -8 degrees, 0.4s ease)
  2. A small lightbulb "💡" appears above Byte's head
     Framer Motion: scale 0 → 1.2 → 1.0, y: -10 → -20, duration 0.5s
  3. Byte holds the tilt for 3 seconds then returns to idle

ON HEART LOST:
  1. Byte turns briefly RED (CSS filter: hue-rotate + saturate)
     Duration: 0.3s red → 0.5s fade back to normal
  2. SAD drop: y: 0 → 8 → 0, 0.8s ease-in-out
  3. Eyes do the worried eyebrow look
  4. Byte settles into sad pose

ON LESSON START (first load):
  1. Byte enters from bottom: y: 80 → 0, opacity 0 → 1

ON STREAK MILESTONE (3-day or 7-day):
  1. Byte does the cheer jump 3 times in succession
```

## CRITICAL RULES (READ BEFORE WRITING ANY LESSON CODE)

1. topicName is injected into EVERY instruction string.
   Use a template function: `(topic: string) => \`...\``
   Every instruction references the kid's topic.

2. startingCode after Step 2 of every lesson ALWAYS contains the topic.
   The kid's topic appears pre-filled. They are building THEIR thing.

3. Validators NEVER check for the topic name.
   We check for HTML structure, not content.
   A kid who writes <h1>I love pizza more than Minecraft</h1> should pass.

4. Every hint is a nudge, not an answer.
   Byte points the kid toward the solution. Never gives it.

5. Step types ROTATE:
   Step 1: write (from scratch)
   Step 2: write/personalize (topic-specific)
   Step 3: identify (read and recognize)
   Step 4: fix (broken code to repair)
   Step 5: combine (use this lesson + a previous lesson)

```typescript
      startingCode: (topic: string) =>
        `<html>\n` +
        `  <head><title>${topic}</title></head>\n` +
        `  <body>\n` +
        `    <h1>${topic}</h1>\n` +
        `    \n` +
        `  </body>\n` +
        `</html>`,
      xp: 10,
      validate: (doc: Document) =>
        doc.querySelector('img[src]') !== null,
    },

    // ── STEP 2: WRITE — Add alt text ────────────────────────────
    {
      id: 'step-2',
      type: 'write',
      instruction: (topic: string) =>
        `Now add a description to your image.\n\n` +
        `This is called alt text — it helps people who cannot see images.\n\n` +

        `for someone who cannot see the image.\n` +
        `It is also what shows when the image fails to load.\n` +
        `Always add alt text. Always.\n` +
        `Describe your ${topic} image.`,
      hint: `Add alt inside the img tag:\n` +
            `<img src="https://..." alt="Description here">\n` +
            `The alt goes INSIDE the same tag as src.\n` +
            `Separate them with a space.`,
      startingCode: (topic: string) =>
        `<html>\n` +
        `  <head><title>${topic}</title></head>\n` +
        `  <body>\n` +
        `    <h1>${topic}</h1>\n` +
        `    <img src="https://picsum.photos/300/200">\n` +
        `  </body>\n` +
        `</html>`,
      xp: 10,
      validate: (doc: Document) =>
        doc.querySelector('img[alt]') !== null &&
        (doc.querySelector('img')?.getAttribute('alt')?.length ?? 0) > 0,
    },

    // ── STEP 3: IDENTIFY — Image size with CSS ──────────────────

      id: 'step-3',
      type: 'identify',
      instruction: () =>

            `  img {\n` +
            `    width: 300px;\n` +

            `</style>\n` +

      startingCode: (topic: string) =>
        `<html>\n` +
        `  <head>\n` +
        `    <title>${topic}</title>\n` +
        `    <style>\n` +
        `      /* Add image width here */\n` +
        `      \n` +
        `    </style>\n` +
        `  </head>\n` +
        `  <body>\n` +
        `    <h1>${topic}</h1>\n` +
        `    <img src="https://picsum.photos/300/200" alt="A picture">\n` +
        `  </body>\n` +
        `</html>`,
      xp: 10,
      validate: (doc: Document) => {
        const img = doc.querySelector('img') as HTMLImageElement | null
        if (!img) return false
        const width = window.getComputedStyle(img).width
        return width !== '' && width !== 'auto'
      },
    },

    // ── STEP 4: FIX — Missing closing quote on src ──────────────
    {
      id: 'step-4',
      type: 'fix',
      instruction: () =>
        `Byte's image is broken — the icon shows but no photo.\n\n` +
        `Find the mistake in the img tag.\n` +
        `Clue: attribute values need quotes around them.`,
      bytePrompt: () =>
        `This is the MOST COMMON image mistake.\n` +
        `Every attribute value needs quotes.\n` +
        `A quote to START the value.\n` +
        `A quote to END the value.\n` +
        `Count the quote marks in the src attribute.`,
      hint: `An attribute value needs TWO quote marks:\n` +
            `src="the url goes here"\n` +
            `One opening " at the start.\n` +
            `One closing " at the end.\n` +
            `If one is missing, the browser cannot read the URL.`,
      startingCode: (topic: string) =>
        `<html>\n` +
        `  <head><title>${topic}</title></head>\n` +
        `  <body>\n` +
        `    <h1>${topic}</h1>\n` +
        `    <img src="https://picsum.photos/300/200 alt="A picture">\n` +
        `  </body>\n` +
        `</html>`,
      xp: 10,
      validate: (doc: Document) =>
        doc.querySelector('img[src]') !== null &&
        doc.querySelector('img[alt]') !== null,
    },

    // ── STEP 5: COMBINE — Second image, full styled page ────────
    {
      id: 'step-5',
      type: 'combine',
      instruction: (topic: string) =>
        `Final step — add a SECOND image!\n\n` +
        `Your ${topic} page should now have:\n` +
        `✓ h1 heading with your topic\n` +
        `✓ At least one paragraph\n` +
        `✓ Two images with src AND alt\n` +
        `✓ CSS giving images a width\n\n` +
        `Check the preview. This is a real webpage.`,
      bytePrompt: (topic: string) =>
        `Two images. Headings. Paragraphs. CSS.\n` +
        `Look at that preview.\n` +
        `That is a REAL webpage about ${topic}.\n` +
        `You built that.`,
      hint: `Just copy your img tag and change the URL:\n` +
            `<img src="https://picsum.photos/300/201" alt="Second image">\n` +
            `(Change the number at the end for a different photo!)`,
      startingCode: (topic: string) =>
        `<html>\n` +
        `  <head>\n` +
        `    <title>${topic}</title>\n` +
        `    <style>\n` +
        `      img { width: 300px; }\n` +
        `    </style>\n` +
        `  </head>\n` +
        `  <body>\n` +
        `    <h1>${topic}</h1>\n` +
        `    <p>Something amazing about ${topic}.</p>\n` +
        `    <img src="https://picsum.photos/300/200" alt="First image">\n` +
        `    \n` +
        `  </body>\n` +
        `</html>`,
      xp: 10,
      validate: (doc: Document) =>
        doc.querySelectorAll('img[src]').length >= 2 &&
        doc.querySelectorAll('img[alt]').length >= 2,
    },
  ],
}
```

## CRITICAL RULES — DO NOT VIOLATE THESE

1. startingCode for FIX and COMBINE steps is a function that takes topicName.
   The editor pre-fills with the kid's actual topic in the code.
   A Minecraft kid sees <h1>Minecraft</h1> in the starting code.
   A dog kid sees <h1>My Dog</h1>.

2. Write steps (step-1) start with empty editor OR the page structure only.
   They do not pre-fill the answer — they pre-fill the SCAFFOLD.

3. Byte never talks about something generic.
   Every bytePrompt references the topic or the specific concept.
   Replace "your page" with "your [topic] page" everywhere.

4. The validator iframe runs inside a sandboxed environment.
   window.getComputedStyle inside the validator refers to the IFRAME's window.
   The validator function receives doc: Document from the iframe.
   Do NOT use the outer window for computed styles.
   Correct pattern: (doc.querySelector('h1') as HTMLElement)?.style.color
   OR pass the iframe contentWindow to getComputedStyle.

5. Lesson data files export typed Lesson objects.
   All fields are either strings or functions returning strings.
   The LessonPage resolves them at render time with topicName.

6. Never autoplay any animation.
   All Byte animations are triggered by events (step load, pass, fail, etc.)
   The only looping animations are the body float and the ambient particles.

7. The preview is NEVER blank.
   Even with empty code: show the shell with empty body.
   Even with invalid code: show whatever parsed.
   The visual feedback loop IS the lesson.
